<p class="text-lg leading-relaxed mb-6">
Most people still “chat” with AI. GPT‑5 doesn’t want a chat. It wants a spec—and then it ships.
</p>

<p class="mb-6">
That one shift flips the game: the teams who learn to spec fast will compound faster than the teams who talk longer.
</p>

<h2 class="text-2xl font-bold mt-10 mb-4">What changed (and why it matters)</h2>
<ul class="list-disc pl-6 space-y-2 mb-6">
  <li><strong>Bias to ship.</strong> GPT‑5 asks at most one clarifying question, then executes. If your prompt has fuzzy assumptions, it will confidently build the wrong thing—quickly.</li>
  <li><strong>Spec &gt; conversation.</strong> The model favors clear deliverables, constraints, and acceptance criteria. Treat it like a capable, literal operator.</li>
  <li><strong>Agentic tools.</strong> Unless you say otherwise, it may search, write code, or invoke tools. You must allow/forbid tools explicitly.</li>
  <li><strong>Memory + workspace.</strong> Persist preferences (tone, length, format) and version work in a single place. Treat ChatGPT like an operating system for your workflows, not a one‑off chat.</li>
  
</ul>

<p class="mb-6">
<em>Translation for owners:</em> if you run a practice or a small team, this is operational leverage. The market rewards speed; GPT‑5 pays interest on action.
</p>

<hr class="my-8" />

<h2 class="text-2xl font-bold mt-10 mb-4">The Spec Mindset (copy‑paste template)</h2>

<p class="mb-4">Use this structure verbatim. It’s dry. That’s the point.</p>

```text
Task:
Deliverable: [format, length, audience]
Assumptions: [context, scope, timelines]
Non‑Goals: [what to exclude]
Tools: [allowed], [forbidden]
Acceptance: [success criteria + decision‑readiness]
```

<p class="mb-6">
Why it works: it constrains “speculative execution,” prevents surprise tool usage, and keeps outputs decision‑ready instead of decorative.
</p>

<hr class="my-8" />

<h2 class="text-2xl font-bold mt-10 mb-4">Failure modes (and the counterplays)</h2>

<h3 class="text-xl font-semibold mt-6 mb-2">1) Speculative overbuild</h3>
<ul class="list-disc pl-6 mb-4">
  <li><strong>Symptom:</strong> You wanted a check; you got a 2,000‑word opus.</li>
  <li><strong>Fix:</strong> Add Non‑Goals and a hard cap in Deliverable (“≤400 words,” “bullets only,” “one table”).</li>
  
</ul>

<h3 class="text-xl font-semibold mt-6 mb-2">2) Tool surprises</h3>
<ul class="list-disc pl-6 mb-4">
  <li><strong>Symptom:</strong> It searched the web or wrote code you didn’t want.</li>
  <li><strong>Fix:</strong> Set Tools (“No web.run. No code. Reason from provided context only.”).</li>
  
</ul>

<h3 class="text-xl font-semibold mt-6 mb-2">3) Compounding wrong assumptions</h3>
<ul class="list-disc pl-6 mb-6">
  <li><strong>Symptom:</strong> A tiny misunderstanding becomes a very polished mistake.</li>
  <li><strong>Fix:</strong> Front‑load Assumptions and Acceptance. Force alignment before output.</li>
</ul>

<p class="italic text-sm mb-8">Pro tip: image generation often suppresses commentary—split “make image” and “analyze image” into separate turns.</p>

<hr class="my-8" />

<h2 class="text-2xl font-bold mt-10 mb-4">Swipe File: 4 Specs Prism’s Clients Can Use Today</h2>

<h3 class="text-xl font-semibold mt-6 mb-2">1) 90‑Day Growth Plan for Google Business Profile (Dental)</h3>
<p class="mb-2"><strong>Task:</strong> Build a 90‑day plan to increase GBP calls and directions for a family dental practice in Palo Alto.</p>
<p class="mb-2"><strong>Deliverable:</strong> One-page brief (≤400 words) with weekly actions + KPI table; audience: dentist-owner.</p>
<p class="mb-2"><strong>Assumptions:</strong> $1.5k/mo budget; services: exams, cleanings, whitening, implants; target: +35% interactions in 90 days.</p>
<p class="mb-2"><strong>Non‑Goals:</strong> No generic SEO theory; no ads setup steps beyond headlines.</p>
<p class="mb-2"><strong>Tools:</strong> No web browsing. Use best practices only.</p>
<p class="mb-4"><strong>Acceptance:</strong> Clear week-by-week checklist, KPI targets, and 3 risks with mitigations.</p>

<h3 class="text-xl font-semibold mt-6 mb-2">2) New‑Patient Landing Page Wireframe</h3>
<p class="mb-2"><strong>Task:</strong> Create a wireframe outline for a high‑converting “New Patient Special” page.</p>
<p class="mb-2"><strong>Deliverable:</strong> Section-by-section bullets + sample copy (≤300 words) + one FAQ list.</p>
<p class="mb-2"><strong>Assumptions:</strong> Mobile-first; social proof available; online booking link exists.</p>
<p class="mb-2"><strong>Non‑Goals:</strong> No dev code. No long-form blog content.</p>
<p class="mb-2"><strong>Tools:</strong> No code. No images. Reason from provided details.</p>
<p class="mb-4"><strong>Acceptance:</strong> Skimmable outline that a designer can build in &lt;1 hour.</p>

<h3 class="text-xl font-semibold mt-6 mb-2">3) Monthly KPI Exec Summary (Owner‑Readable)</h3>
<p class="mb-2"><strong>Task:</strong> Turn raw analytics (visits, calls, form fills, booked patients) into an exec summary.</p>
<p class="mb-2"><strong>Deliverable:</strong> 6-bullet summary + 1 table + 3 recommendations; audience: owner.</p>
<p class="mb-2"><strong>Assumptions:</strong> Focus on decision-ready insights; compare to prior month and 3‑month trend.</p>
<p class="mb-2"><strong>Non‑Goals:</strong> No jargon. No vanity metrics.</p>
<p class="mb-2"><strong>Tools:</strong> No browsing. Use the numbers I paste next.</p>
<p class="mb-4"><strong>Acceptance:</strong> Owner can decide next action in &lt;2 minutes of reading.</p>

<h3 class="text-xl font-semibold mt-6 mb-2">4) 30‑Post Content Calendar That Actually Sells</h3>
<p class="mb-2"><strong>Task:</strong> Plan 30 posts that convert browsers into booked appointments.</p>
<p class="mb-2"><strong>Deliverable:</strong> Table with date, hook, angle, CTA, asset type (reel/static), and why it works.</p>
<p class="mb-2"><strong>Assumptions:</strong> Services: whitening, Invisalign, implants; tone: friendly + premium; local radius: 10 miles.</p>
<p class="mb-2"><strong>Non‑Goals:</strong> No generic holiday posts. No AI stock images.</p>
<p class="mb-2"><strong>Tools:</strong> No browsing.</p>
<p class="mb-6"><strong>Acceptance:</strong> At least 10 posts tie to a bookable offer with measurable CTA.</p>

<hr class="my-8" />

<h2 class="text-2xl font-bold mt-10 mb-4">One‑Week Rollout Plan (simple, aggressive)</h2>
<ul class="list-disc pl-6 space-y-1 mb-6">
  <li><strong>Day 1–2:</strong> Pick your highest‑volume AI task (reports, pages, calendars). Write the Spec v1 using the template.</li>
  <li><strong>Day 3:</strong> Encode memory: preferred tone, word count, and formats (e.g., “3‑bullet exec summaries”).</li>
  <li><strong>Day 4–5:</strong> Run 3 cycles: spec → output → acceptance check → tighten spec. Save the winner as a prompt asset.</li>
  <li><strong>Day 6:</strong> Standardize handoff: where outputs live, who approves, definition of “done.”</li>
  <li><strong>Day 7:</strong> Measure: cycle time saved, approvals on first pass, booked appointments or qualified leads created.</li>
</ul>

<p class="mb-6">Speed is a habit. Specs make speed safe.</p>
