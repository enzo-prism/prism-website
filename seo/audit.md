# Prism SEO Audit (Repo Inventory)

Framework detected: **Next.js 15 App Router** (`app/` directory, `export const metadata`, `generateMetadata`, `app/sitemap.ts`, `app/robots.ts`).

Inventory source: `seo/inventory.csv` generated by `pnpm seo:inventory`.

## Top issues (priority order)

1. **Canonical tags missing on ~36 indexable routes.**  
   - Mostly case studies, some marketing pages (e.g. `/ai`, `/book-a-shoot`, many `/case-studies/*`).  
   - **Fix:** Add `alternates: { canonical: "https://www.design-prism.com/<route>" }` (or a relative canonical if you prefer, relying on `metadataBase`) to every public route you want indexed.

2. **Some canonicals are relative and inconsistent.**  
   - `/`, `/case-studies/dr-christopher-wong`, `/case-studies/mataria-dental-group`, `/case-studies/olympic-bootworks` use `/…` canonicals.  
   - **Fix:** Standardize to absolute `https://www.design-prism.com/...` for clarity and to avoid edge cases in crawlers.

3. **`/ai-website-launch` has no `metadata` export.**  
   - Title/description/canonical are empty.  
   - **Fix:** Add `export const metadata` in `app/ai-website-launch/page.tsx`.

4. **Checkout thank-you pages lack metadata and should not be indexed.**  
   - `/checkout/grow/thank-you`, `/checkout/launch/thank-you`, `/checkout/scale/thank-you`.  
   - **Fix:** Add `robots: { index: false, follow: false }` and minimal metadata or redirect to a single `/thank-you`.

5. **Other confirmation pages are indexable but should be `noindex`.**  
   - Examples: `/analysis-thank-you`, `/pricing/thank-you`, `/book-a-shoot/thank-you`, `/thanks`, `/thank-you`. Only `/thanks-call` is `noindex`.  
   - **Fix:** Add `robots: { index: false, follow: false }` to all post‑conversion pages.

6. **Homepage lacks a clear visible H1.**  
   - Inventory did not detect an H1 on `/`.  
   - **Fix:** Ensure the hero renders a single `<h1>` describing Prism’s primary offer.

7. **Several case studies do not clearly expose a visible H1 at the top.**  
   - Many `/case-studies/*` pages don’t include an H1 in their local route files.  
   - **Fix:** Add a top‑level `<h1>` in each case study hero (or ensure the imported client page includes one).

8. **Structured data missing on ~77 routes.**  
   - Especially `/about`, `/blog` index, `/services`, `/story`, `/success-stories`, most “why‑love‑prism” pages.  
   - **Fix:** Add lightweight JSON‑LD where appropriate:  
     - `Organization` + `Person` for `/about`  
     - `CollectionPage`/`Blog` for `/blog` index  
     - `Service` for `/services` and service category pages  
     - `Article`/`Review` aggregates for proof pages.

9. **Key titles exceed recommended SERP length (>60 chars).**  
   - `/case-studies/olympic-bootworks`, `/case-studies/mataria-dental-group`, `/google`, `/google/dental-ads`, `/replit`.  
   - **Fix:** Shorten titles to ~50–60 chars while keeping primary keyword first.

10. **Blog meta descriptions are frequently too long (>160 chars).**  
    - 21 posts exceed length; SERPs truncate.  
    - **Fix:** Rewrite frontmatter `description` to 140–160 chars with the main benefit + keyword.

11. **18 blog posts lack an explicit `image` frontmatter field.**  
    - Not fatal (OG images fall back to `/api/og/blog/...`), but limits social previews and hero images.  
    - **Fix:** Add `image: "/blog/<slug>.png"` (or OG API URL) to each missing post.

12. **Many case studies have sparse Open Graph config.**  
    - Several pages only set title/description and omit images/type.  
    - **Fix:** Add `openGraph.images` + `openGraph.type="article"` to each case study metadata export.

13. **`/blog` index has no structured data or canonical collection metadata.**  
    - **Fix:** Add `alternates.canonical` + `Blog`/`CollectionPage` JSON‑LD.

14. **Inconsistent category casing in blog frontmatter.**  
    - Categories vary (`AI & Dentistry`, `Business & AI`, etc.).  
    - **Fix:** Standardize taxonomy for better internal linking and archive UX.

15. **Large public images likely harming CWV and SEO.**  
    - `public/` contains many 3–4MB PNG heroes.  
    - **Fix:** Convert top‑traffic hero assets to WebP/AVIF, set correct `sizes`, and ensure LCP images are optimized.

16. **Video embeds load early on multiple pages.**  
    - Hero YouTube/Vimeo iframes may delay LCP/TTI.  
    - **Fix:** Use click‑to‑load / lite embeds and lazy‑load below‑the‑fold videos.

17. **Some routes still rely on layout defaults for OG/Twitter.**  
    - Missing route‑level OG images means generic previews.  
    - **Fix:** Add per‑route OG images for high‑value pages.

18. **No explicit robots meta on most pages.**  
    - Defaults to index/follow, but makes intent unclear.  
    - **Fix:** Add explicit `robots` on any page that is *not* intended for indexing; keep index pages default.

19. **Internal link hubs are thin on some service pages.**  
    - Service pages could cross‑link more deeply to case studies and blog clusters.  
    - **Fix:** Add “related case studies” and “related posts” blocks per service.

20. **Sitemap/robots are present but not budgeted for coverage.**  
    - With 149 routes, ensure the sitemap includes only indexable ones.  
    - **Fix:** Add a filter in `app/sitemap.ts` for noindex routes.
